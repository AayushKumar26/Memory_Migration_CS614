Ring buffer records analysed:
  Standard path (rocm_path=0): 8192 records
  ROCM path     (rocm_path=1): 8192 records

Pipeline structure:
─────────────────────────────────────────────────────────────────────
  Standard:  Lock → [Unmap → Copy → Remap] → Unlock
                     ↑ thread sleeps here ↑
                     migration entry (Present=0) blocks faulting threads
                     stall = unmap_ns + copy_ns + remap_ns

  ROCM:      Lock → Copy → [PTE swap] → Unlock
                            ↑ stall ↑
                            no migration entry installed
                            faulting thread retries (no sleep)
                            stall = ro_swap_ns only
─────────────────────────────────────────────────────────────────────

Application-visible stall window (primary metric):
─────────────────────────────────────────────────────────────────────
  Standard  stall = unmap + copy + remap
                  = 842 + 786 + 107 = 1735 ns
  ROCM      stall = ro_swap_ns (PTE swap window only)
                  = 1195 ns
  Reduction : 1735 ns → 1195 ns  =  31.1%  (1.45× faster)
  Projected : 15× (from Benchmark A quiescent baseline)
  Actual    : 1.45× (under live system load, single TLBI per page)

Stall window visual (each # ≈ 43 ns):
  Standard  [########################################]  1735 ns  (unmap+copy+remap)
  ROCM      [###########################.............]  1195 ns  (ro_swap only)

ROCM PTE swap detail (ro_swap_ns):
─────────────────────────────────────────────────────────────────────
  Mean : 1195 ns ± 10220 ns
  Why not 70 ns as projected:
    Projected 70 ns was based on Remap stage (install PTE into
    a non-present slot — no TLBI needed).  ROCM replaces a live
    translation, requiring TLBI VAE1IS + DSB ISH (~420-1000 ns).
    The projection was for a different operation.  The correct
    lower bound is one TLBI round-trip on ARM64 (~420 ns).
─────────────────────────────────────────────────────────────────────

Key findings:
  1. Application stall:  1735 ns (standard) → 1195 ns (ROCM)
                         1.45× reduction  (projected 15×)
  2. Remap stage:        107 ns (standard) → 0 ns (ROCM)
                         remove_migration_ptes() fully eliminated
  3. rocm_path=1 records: 8192 / 8192  (100%)
  4. ro_swap_ns mean:    1195 ns (TLBI VAE1IS + DSB ISH per page)
  5. Copy during ROCM:   888 ns — occurs while src PTEs are live,
                         no thread stalls during this copy

Conclusion:
  ROCM successfully eliminates the migration-entry stall for read-only
  file-backed pages.  The application-visible stall shrinks from
  1735 ns (standard: Unmap+Copy+Remap) to 1195 ns (ROCM: PTE swap only),
  a 1.45× reduction measured directly from the kernel ring buffer.
  The projected 15× was an underestimate of TLBI cost; the actual
  ROCM stall is bounded by one TLBI VAE1IS round-trip (~420-1000 ns)
  rather than zero.  Remap (remove_migration_ptes) is fully eliminated.
  The data-copy phase moves to before the stall window, at full NEON
  bandwidth with no competing waiters.
